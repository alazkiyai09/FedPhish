## Review Framework

For each project, the following checks are performed:

### 1. Requirements Compliance
- All requirements implemented
- Exact function signatures and return types
- Specification accuracy

### 2. Code Quality
- Type hints on all functions
- Docstrings on public functions
- No hardcoded values
- Proper error handling
- No code duplication
- Clear naming
- Organized imports

### 3. Bugs & Edge Cases
- Division by zero risks
- Empty array/list handling
- None/null checks
- Index out of bounds
- Type mismatches
- Race conditions
- Memory leaks

### 4. Performance
- Unnecessary loops
- Repeated computations
- Large data copies
- Inefficient data structures

### 5. Security
- Input validation
- No sensitive data in logs
- Proper cryptographic use
- No hardcoded secrets

### 6. Testability
- Small focused functions
- Mockable dependencies
- Minimized side effects
- Main block demonstrations
